Hexo:基于nodejs的博客框架
hexo
hexo
Publish Date:   2019-07-08
Read Count:  

Hexo 是一个基于nodejs的博客框架，快速简洁高效，是他的特点。本文主要是我在安装使用部分的体验，官网 hexo.io 有完整的文档可以使用。相比与2017刚刚接触的时候，体验顺畅了很多。总结起来，为什么又新开了一个博客，又开始使用hexo,主要还是想把从事前端的这三年做一些总结和梳理，使用hexo就是因为他够简单快速。

这个博客是依托在GitHub上的。目前访问的链接也是gitpage

Github

git 是软件工程师必备的的技能之一。关于git的介绍这里不在赘述。以下内容基于你有一定的技术能力储备（git、node等）。
当你依照官网的步骤在本地运行起 hexo server 也许你看到的是以下内容。对了，我这里都有以mac平台为基础。

~/github/hexoblog » yarn dev                                                                                             zhuxiaopeng@zhuxiaopengdeMacBook-Pro
yarn run v1.5.1
warning ../../package.json: No license field
$ hexo g && hexo server
INFO  Start processing
INFO  Files loaded in 274 ms
INFO  Generated: 2019/07/08/Hexo-基于nodejs的博客框架/index.html
INFO  Generated: index.html
INFO  2 files generated in 592 ms
INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

这里你看到的 yarn dev 是我在package.json中写了以下内容

"scripts": {
    "dev": "hexo g && hexo server",
    "deploy": "hexo clean && hexo deploy"
}

yarn 类似于npm 这里也不展开。你可以使用npm。我注意执行 hexo init 之后，hexo 会使用yarn安装默认的依赖。并在项目目录中生成yarn.lock文件。

根目录下 _config.yml 是对生成静态文件的一个配置文件。yml 语法可以自行了解。

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: https://orangeborning.github.io/ # 网址
root: / # 网站根目录
permalink: :year/:month/:day/:title/ # 文章的 永久链接 格式
permalink_defaults: # 永久链接中各部分的默认值    

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: https://github.com/OrangeBorning/OrangeBorning.github.io.git #库（Repository）地址
  branch: master #publ分支名称。如果您使用的是 GitHub 或 GitCafe 的话，程序会尝试自动检测。ished
  # message: [message] 自定义提交信息 (默认为 Site updated: {{ now('YYYY-MM-DD HH:mm:ss') }})

这里要注意将 deploy 部分换成你的github仓库地址。URL部分，根目录建议使用默认的。

使用官网文档可以很顺畅的完成这里的操作。

Themes

官方默认给你安装了一个主题，主题在 /themes 下，如果你和我一样不喜欢他的样子，你可以在主题市场里找找看有没有喜欢的。当然你也可以自己写一个主题。
安装一个新的 theme 你可以在市场找到合适的后，点击你喜欢的主题的名称前往他的 github 地址。在你的项目根目录下运行

git clone https://github.com/kinggozhang/hexo-theme-ace.git themes/ace

这样就会安装你需要的主题。

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: ace # landscape # 当前主题名称。值为false时禁用主题

修改配置文件下的 theme 字段内容为你安装到 /themes/ 下到目录名

不同的主题下作者有提供自定义的配置，你可以在主题下找到另一个 _config.yml 这个配置详情看对应主题的文档

域名

使用自己的域名需要你首先得有一个域名，具体购买配置可以自行搜索。然后通过CNAME将域名映射到你的gitpage
